# Skill: issue-06-local-storage-foundation

## Issue

- GitHub Issue: #6 データベース

## 目的

`Dexie.js`（IndexedDB）を使って、チャット履歴・メモ・マーカー・サイト単位のメタ情報を一貫した構造で保存できる基盤を作る。

## 重要度

- 優先度: **P0**
- 理由: 保存が必要な複数Issueの前提機能であるため。

## スコープ

- DexieのDB名・テーブル・インデックス設計
- 読み書きAPIの抽象化
- URL/日時/本文を持つ保存モデルの定義
- サイト・メモのタグ付け設計（付与/解除/検索）
- データ破損を防ぐ最低限のバリデーション
- 将来スキーマ変更に備えたマイグレーション方針

## 機能要件

1. `Dexie.js` を用いた永続化レイヤーを `core/` に設ける。
2. チャット履歴を `url`, `title`, `createdAt`, `updatedAt`, `messages[]` 単位で保存できる。
3. メモを `siteId`, `noteId`, `markdown`, `sourceLinks[]`, `createdAt`, `updatedAt` で保存できる。
4. マーカーを `siteId`, `markerId`, `color`, `rangeDescriptor` または DOM復元情報で保存できる。
5. 将来のライブラリー機能を見据え、サイト単位の集約オブジェクトを持てる。
6. 保存・取得・更新・削除の基本操作を共通APIとして提供する。
7. 主要検索パス（`siteId`, `updatedAt`, `url`）にインデックスを定義する。
8. `chrome.storage.local` にはAPIキーやUI設定のみ保存し、ドメインデータはDexieへ分離する。
9. サイトとメモにタグを複数付与できる（`tags[]`）。
10. タグの付与・解除・名称変更・正規化（前後空白除去、大小文字ポリシー統一）を行える。
11. タグでサイト/メモを絞り込める検索APIを提供する。

## 非機能要件

- Manifest V3 の制約下で同期的な重処理を避ける。
- 既存の翻訳/AI機能に副作用を出さない。
- スキーマ変更時に拡張しやすい構造にする。
- DBバージョン管理（Dexie `version()`）と段階的マイグレーションを行える。

## データモデル要件

- Dexieテーブルは少なくとも `sites`, `chats`, `notes`, `markers` を持つ。
- 各レコードに `schemaVersion` または互換管理用の版情報を保持する。
- `siteId` は URL を基に安定生成する。
- メッセージは `role`, `content`, `timestamp` を最低限保持する。
- 将来の検索を見据え、本文はプレーンテキスト抽出値も持てる形にする。
- `sites` から `notes` / `markers` / `chats` を参照できるよう外部キー相当（`siteId`）を統一する。
- `sites.tags[]`, `notes.tags[]` を保持し、タグ検索用の補助インデックス（必要ならタグ中間テーブル）を持てる。
- タグは重複を許容せず、保存時に正規化する。

## 受け入れ条件

- 任意のサイトに対してチャット履歴を保存・再取得できる。
- 任意のサイトに対してメモ配列を保存・更新できる。
- マーカー情報を色付きで保存・削除できる。
- IndexedDB未初期化状態でも例外なく初期化できる。
- 主要クエリ（`siteId` 指定取得）が実用速度で動作する。
- サイト・メモのタグ付与/解除/更新ができる。
- 1つ以上のタグ条件でサイト・メモを絞り込み取得できる。

## 変更候補

- `core/` に Dexieラッパーとリポジトリ層を追加
- `core/background.js` から保存処理を呼べるようにする
- 必要に応じて `features/ai/` と `features/translate/` から共通APIを利用する
- `vendor/` に Dexie を配置し、Manifest/読み込み経路を整備する

## 実装順

1. DexieのDBスキーマ（テーブル/インデックス/バージョン）を決める。
2. `core/` にDexie初期化とリポジトリAPIを追加する。
3. チャット保存を最初に接続し、既存機能への影響を検証する。
4. メモ/マーカー保存へ段階的に適用する。
5. 設定データは従来どおり `chrome.storage.local` に残し、責務分離を確認する。
6. サイト/メモのタグ付けAPIとタグ検索APIを追加し、将来のライブラリーUIに備える。
