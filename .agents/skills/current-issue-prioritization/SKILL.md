# Skill: current-issue-prioritization

## 対象Issue

- #6 データベース
- #5 メモ機能の追加
- #8 ウェブサイトマーカー機能
- #9 ウェブサイト管理ライブラリー
- #7 インターネット検索機能

## 優先順位

1. **P0**: #6 データベース
2. **P1**: #5 メモ機能の追加
3. **P1**: #8 ウェブサイトマーカー機能
4. **P2**: #9 ウェブサイト管理ライブラリー
5. **P2**: #7 インターネット検索機能

## 判断理由

- #6 は #5, #8, #9 の保存基盤になり、先に設計しないとデータ構造がぶれる。
- #5 は既存のAIチャットと接続でき、ユーザー価値が高い。
- #8 はDOM座標保存設計が必要で、#6 の保存仕様に依存する。
- #9 は保存対象が増えた後に情報設計すると無駄が少ない。
- #7 は既存AI基盤を拡張できる一方、検索プロバイダ・引用形式・権限の未確定要素が多い。

## 実行順

1. ローカルストレージDBのスキーマとアクセサを定義する。
2. メモ機能を追加し、AI回答からメモ化できるようにする。
3. ウェブサイトマーカー機能を追加する。
4. 保存済みサイト/メモを束ねるライブラリーUIを設計する。
5. 外部検索トグルとソース表示をAI回答フローに統合する。

## 適用ルール

- 依存Issueを飛ばして後続Issueを先行実装しない。
- 各Issueは既存の翻訳機能とAI機能を壊さない範囲で分離実装する。
- 保存系データは `chrome.storage.local` を第一選択とし、構造の互換性を維持する。
- 新規UIは side panel に段階的に追加し、既存ワークスペース切替と競合させない。
