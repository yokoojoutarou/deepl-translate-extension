# 02: GitHub CLI実行コンテキスト確認

## 目的

`gh` コマンド実行時の認証・対象リポジトリ不一致を防止する。

## 入力

- 01で確定した `owner`, `repo`

## 処理

1. `gh auth status` で認証状態を確認。
2. `gh repo view <owner>/<repo>` でアクセス可否を確認。
3. ローカルGitリポジトリとIssueの対象リポジトリが一致するか確認。
4. 権限不足（read/write不足）時は操作可能範囲を明示。

## 完了条件

- `gh` が対象Issueを読み取り可能。
- リポジトリ不一致や認証不備を事前検出済み。
