# Skill: issue-05-note-feature

## Issue

- GitHub Issue: #5 メモ機能の追加

## 目的

サイドパネルを離れずに、閲覧中サイトに紐づくMarkdownメモを作成・保存・再利用できるようにする。

## 重要度

- 優先度: **P1**
- 依存: #6 データベース

## スコープ

- メモ入力UI
- Markdown表示
- AI回答のメモ取り込み
- ページ内参照リンクの保存
- サイト単位のメモ保存

## 機能要件

1. side panel にメモワークスペースを追加する。
2. メモ本文は Markdown で入力・表示できる。
3. AI回答をワンクリックでメモへ追記または新規作成できる。
4. メモは現在のページURLまたは `siteId` に紐づけて保存する。
5. メモ内にページ内参照リンクを保持できる。
6. 保存済みメモを一覧表示し、選択・編集・削除できる。

## ページ連携要件

- 選択テキストまたはマーカー範囲をメモへ引用できる。
- 引用には、復元可能なDOM位置情報またはアンカー情報を保持する。
- リンク押下時に対象箇所へスクロールできる。

## UI要件

- 既存の Translate / AI と競合しない形でモード追加またはタブ追加する。
- 入力中に自動保存するか、保存ボタンを明示する。
- Markdownプレビューは既存チャット描画資産を可能な限り再利用する。

## 非機能要件

- メモ未保存でもUIが壊れない。
- 大量メモでも最低限の操作性を保つ。
- 危険なHTMLは描画しない。

## 受け入れ条件

- ページごとにメモを新規作成・保存・再編集できる。
- AI回答をメモへ転記できる。
- メモ内リンクから元ページの該当箇所へ移動できる。
- 再読込後も保存済みメモが復元される。

## 変更候補

- `features/notes/` または `features/ai/sidepanel/` 配下への新規UI追加
- `core/background.js` の保存連携
- `features/translate/content.js` もしくは新規 content script による位置復元支援

## 実装順

1. 保存モデルとメモ一覧取得をつなぐ。
2. メモエディタと一覧UIを追加する。
3. AI回答の取り込み導線を追加する。
4. ページ内リンク復元を実装する。
