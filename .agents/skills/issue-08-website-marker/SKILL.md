# Skill: issue-08-website-marker

## Issue

- GitHub Issue: #8 ウェブサイトマーカー機能

## 目的

閲覧中ページのテキスト範囲を3色でハイライトし、再訪時に復元できるようにする。

## 重要度

- 優先度: **P1**
- 依存: #6 データベース

## スコープ

- テキスト選択からのハイライト作成
- 3色切替
- DOM位置保存
- 再描画と削除

## 機能要件

1. 選択中テキストに対して黄色・赤・青の3色マーカーを適用できる。
2. マーカーごとに `markerId`, `siteId`, `color`, `textQuote`, `rangeDescriptor` を保存する。
3. ページ再訪時に保存済みマーカーを再描画できる。
4. マーカーの削除・色変更ができる。
5. メモ機能と連携できるよう、マーカー参照情報をメモへ渡せる。

## DOM保存要件

- Range直列化だけでなく、復元失敗に備えて `textQuote` を併用する。
- 軽微なDOM変化時も可能な限り復元できる設計にする。
- script/style 領域や編集不能領域へのマーキングは避ける。

## UI要件

- 選択直後に小さなカラーパレットを出す、またはサイドパネル経由で色を選ぶ。
- マーカー一覧からジャンプ・削除できる。
- 既存の選択→翻訳フローを阻害しない。

## 非機能要件

- ページ描画を著しく遅くしない。
- 複数マーカーが重なっても破綻しにくい。
- side panel 未起動時もcontent script単体で最低限動作できる。

## 受け入れ条件

- 任意の選択範囲を3色のいずれかで保存できる。
- ページ再読込後に保存済みマーカーを再表示できる。
- マーカーを削除すると保存データと表示の両方が消える。
- マーカー参照を別機能へ受け渡せる。

## 変更候補

- `features/translate/content.js` の拡張、または `features/marker/` の新設
- `core/background.js` の保存・取得中継
- side panel へのマーカー一覧UI追加

## 実装順

1. Range保存方式を決める。
2. content script で作成/削除を実装する。
3. 再描画フローを追加する。
4. メモ連携用の参照データを公開する。
