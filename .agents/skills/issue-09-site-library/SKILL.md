# Skill: issue-09-site-library

## Issue

- GitHub Issue: #9 ウェブサイト管理ライブラリー

## 目的

保存したサイト・メモ・将来のマーカーを、フォルダ階層で整理し再利用できるライブラリーを提供する。

## 重要度

- 優先度: **P2**
- 依存: #6 データベース, #5 メモ機能の追加

## スコープ

- フォルダ構造
- サイトフォルダ管理
- メモ紐づけ
- サイドパネルまたは専用ビューでの一覧表示

## 機能要件

1. ユーザーが任意のフォルダを作成・名称変更・削除できる。
2. サイト単位の保存単位（サイトフォルダ）を持てる。
3. サイトフォルダには URL, title, notes, markers などを紐づけられる。
4. サイトフォルダを任意フォルダへ移動できる。
5. フォルダ配下のサイトとメモを一覧・検索できる。

## 情報設計要件

- ルートフォルダ配下に任意フォルダと未分類サイトを共存可能にする。
- サイトフォルダは URLベースで一意性を保つ。
- 将来のエクスポートや同期を見据えて、木構造と実データ参照を分離する。

## UI要件

- Zoteroライクな左ペイン階層 + 右ペイン詳細、または段階的な簡易一覧から始める。
- 既存の翻訳/AIパネルとは分離し、誤操作を避ける。
- 最初の実装では最小の一覧/移動/閲覧に絞る。

## 非機能要件

- 階層編集でデータ整合性を崩さない。
- 大量データでも最低限の一覧操作を維持する。
- 保存スキーマを #6 と整合させる。

## 受け入れ条件

- フォルダを作成し、サイトフォルダを格納できる。
- 保存済みメモをサイト単位で閲覧できる。
- フォルダ移動後も参照が壊れない。
- ページ再読込後も階層構造が復元される。

## 変更候補

- `features/library/` の新設
- 保存基盤のフォルダ/ツリー対応
- side panel または別 viewの一覧UI追加

## 実装順

1. フォルダとサイトフォルダのスキーマを追加する。
2. 最小一覧UIを作る。
3. 移動・編集操作を追加する。
4. メモ/マーカー表示を統合する。
