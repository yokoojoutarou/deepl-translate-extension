# Skill: issue-07-web-search

## Issue

- GitHub Issue: #7 インターネット検索機能

## 目的

AIチャットで、閲覧中ページの文脈に加えて外部Web検索結果を参照した回答を生成できるようにする。

## 重要度

- 優先度: **P2**
- 依存: 既存AIチャット基盤
- 注意: 検索プロバイダ未確定のため、設計先行で進める。

## スコープ

- AI送信前の検索有効/無効トグル
- 検索結果取得の抽象化
- 回答末尾のソース一覧表示
- ページ文脈と外部文脈の混在ルール

## 機能要件

1. メッセージ送信ボタン左に検索トグルを置く。
2. トグルON時のみ外部検索を実行する。
3. AIプロンプトには、ページ文脈と検索結果を区別して渡す。
4. 回答表示の末尾に参照ソース一覧を出す。
5. 検索失敗時はページ文脈のみで続行できる。

## 設計要件

- 検索プロバイダ依存を `core/` で抽象化する。
- `AI_ASK` とは別に検索付き経路を追加するか、オプション引数で分岐する。
- ソースは `title`, `url`, `snippet` を最低限持つ。
- プロンプトでは「ページ情報」と「外部検索結果」を明示的に分離する。

## UI要件

- トグル状態が視認できる。
- ソース一覧はクリック可能なリンクとして表示する。
- チャット画面の縦幅を圧迫しすぎない。

## 非機能要件

- APIキーや検索先URLをハードコードしない。
- レート制限や失敗時のフォールバックを持つ。
- 参照元不明の回答を避ける。

## 受け入れ条件

- トグルON/OFFで検索実行有無を切り替えられる。
- 検索利用時の回答末尾にソース一覧が表示される。
- 検索失敗時もAIチャット自体は継続利用できる。
- 既存のページ文脈付きAI回答を壊さない。

## 変更候補

- `features/ai/sidepanel/ai-chat.js`
- `core/background.js`
- 必要なら `manifest.json` の権限追加

## 実装順

1. 検索プロバイダ抽象インターフェースを定義する。
2. UIトグルを追加する。
3. background 側で検索 + AI統合フローを追加する。
4. 回答末尾にソース表示を加える。
